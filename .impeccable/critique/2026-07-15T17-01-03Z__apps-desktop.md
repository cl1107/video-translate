## Anti-Patterns Verdict

**LLM assessment:** 主工作台（Main + Uploader + TaskList）接近 Quiet Workbench；Setup 是典型 SaaS 开场（蓝紫渐变、emoji 三列、text-4xl）；Settings/Logs 硬编码灰与 pastel 色块，与 token 体系断裂。整体「半成品 Electron + 默认 shadcn + 一页模板欢迎」。

**Deterministic scan:** CLI `detect.mjs` 对 21 个 scannable 文件 **0 findings**（exit 0）。浏览器注入 detect.js 发现 **2** 条：`skipped-heading`（setup.tsx h1→h3 跳级）、`layout-transition`（进度条 `transition-[width]`，倾向误报/可接受）。

**Visual overlays:** 评估时在 `http://127.0.0.1:4927/` 注入成功；会话结束已关闭浏览器与 live-server，用户侧当前无持续 [Human] 叠加层。

## Overall Impression

流水线骨架对：上传主路径利落、任务态完整。最大机会不是再堆组件，而是 **统一视觉系统 + 重做开场 + 把完成态钉在「导出可用字幕」**。

## What's Working

1. **MainScreen IA** — sticky 顶栏、分段导航、设置降级、无英雄字号，对齐 Quiet Workbench。
2. **VideoUploader** — 本地为主、链接为次，格式/上限/平台字幕优先写清。
3. **TaskList** — 空状态可行动；暂停/重试/查看/补烧形成真实流水线。

## Priority Issues

### [P0] Setup 开场是 anti-reference 样板
- **What:** `setup.tsx` 蓝紫渐变、emoji 三列、大标题、蓝 CTA
- **Why:** 第一印象违背 PRODUCT anti-references 与 north star
- **Fix:** 欢迎与依赖检查合并为安静门禁；全 token；去 emoji/渐变
- **Suggested command:** `$impeccable onboard apps/desktop`（或 polish setup）

### [P0] 设计系统双轨
- **What:** 主屏 token vs Settings gray-* vs TaskLogs 彩虹 vs Setup 蓝
- **Why:** Consistency 垫底；dark/品牌连续必裂
- **Fix:** 全部语义 token；日志 mono 时间线；设置顶栏复用 Main 语言
- **Suggested command:** `$impeccable colorize apps/desktop` + polish 跨屏统一

### [P1] 成功终点未服务「导出可用字幕」
- **What:** 完成 CTA 是「查看字幕/视频/结果」，无路径/格式摘要
- **Why:** peak-end 偏离 PRODUCT 成功标准
- **Fix:** 主 CTA「打开字幕」+「打开所在文件夹」；产物标签；短 toast
- **Suggested command:** `$impeccable clarify` / polish task complete

### [P1] Settings 认知墙
- **What:** 单页堆 ASR/Ollama/输出/BYOK/烧录/依赖/缓存/危险区；语言 11–12 项
- **Why:** 学习者不可达；违反 progressive disclosure
- **Fix:** 分区 + 默认最小集；常用语言；改即存或脏状态
- **Suggested command:** `$impeccable distill` / layout settings

### [P2] 帮助与恢复空壳
- **What:** 安装指南 console.log；alert()；上传失败不链到设置
- **Why:** 「错误也可行动」失效
- **Fix:** 真链接/步骤；应用内 toast/dialog；失败引导
- **Suggested command:** `$impeccable harden apps/desktop`
